Optimisation

The main element that can relatively affect the loading time of the page is the size of the images you use. To address this I downloaded the same image, say, background image, in different qualities. But the time of loading of the images is pretty fast, on the vast majority of devices. Even on the low-end mobile (tested via browser's dev tools) it takes a small portion of the time compared to the rest. Therefore, I decided to just stick to the default images.
However, when I was using large background image (10mb and more) I expirienced slightly longer loading times, so I switched it to the smaller resolution.

Debugging

W3C Web Validator has beem used to validate both HTML & CSS. I encountered an issue - I used "a" tag inside a button instead of using "onclick" within the button. That is considered a bad practice, therefore, I had to completely remake the HTML & CSS for the navigation menu.

Fortunately, I have not encountered other issues.

Browser testing

To my knowledge, the website looks the same when using dev tools of Chrome, Firefox, and Microsoft Edge browsers.

Accessibility

WAVE Evaluation Tool has been used to test the accessibility. I have not encountered a single issue. However, the evaluation tool counts my QUIZ-related black boxes (on the About page) that reveal the answers after you hover. I do not think that is an issue, but I think it is important to consider this if you need to meet a strict zero issues criteria.
